# 模块详细设计 — 用户故事与交互闭环

> 本文档补充各模块的详细设计，以用户故事串联操作流程，确保每个操作可闭环。覆盖所有参数和交互细节。

---

## 1. 芯片管理模块

### 用户故事

> 作为验证工程师，我要注册一块 Intel Xeon CPU 芯片，填写完整的硬件参数（指令集、代系、型号），以便后续创建针对这块芯片的评测任务。

### 操作闭环

1. 用户进入「芯片管理」页面
2. 点击「注册芯片」按钮
3. 填写表单：
   - 芯片名称（必填，如 "Intel Xeon Platinum 8269CY"）
   - 芯片类型（必填，下拉：CPU/GPU/NPU/TPU/FPGA/ASIC/OTHER）
   - 厂商（必填，如 "Intel"）
   - 架构/指令集（选填，如 "x86_64"、"CUDA"、"ARM"）
   - 代系（选填，如 "Cascade Lake"、"Ada Lovelace"）
   - 具体型号（选填，如 "Platinum 8269CY"、"L40S"）
   - 技术规格 JSON（选填，如 cores、frequency、cache 等）
   - 备注、标签
4. 提交 → POST /api/chips → 返回芯片 ID
5. 芯片列表刷新，新芯片状态为 REGISTERED

### API 详细参数

**POST /api/chips**

```json
{
  "name": "Intel Xeon Platinum 8269CY",    // 必填 string(255)
  "chipType": "CPU",                        // 必填 enum: GPU/NPU/TPU/CPU/FPGA/ASIC/OTHER
  "manufacturer": "Intel",                  // 必填 string(255)
  "architecture": "x86_64",                 // 选填 string(100)
  "generation": "Cascade Lake",             // 选填 string(100)
  "modelName": "Platinum 8269CY",           // 选填 string(200)
  "techSpec": {"cores": 4, "frequency": "2.50GHz"},  // 选填 JSONB
  "remark": "开发测试用",                    // 选填 string(255)
  "tags": "cpu,dev"                          // 选填 string(255)
}
```

**响应：** `{"code": 0, "data": {id, chipNo, name, status: "REGISTERED", ...}}`

## 2. 评测模板模块

### 用户故事 A：查看预置模板

> 作为验证工程师，我要查看"算子性能基准测试"模板的详细配置（测哪些算子、迭代次数、数据精度），确认是否满足我的需求。

### 操作闭环

1. 进入「评测模板」页面，看到 15 个预置 + N 个自定义模板
2. 点击「查看详情」眼睛图标
3. 弹出详情 Modal，展示（和编辑表单完全一致的 Form，所有字段 disabled）：
   - 模板名称、类型标识（🔒系统/自定义）
   - 描述
   - 评测类型（性能评测/精度评测/...）、评测层级（OPERATOR/MODEL/CHIP/COMPARISON）
   - 算子列表（根据 evaluationLayer 联动显隐）
   - 模型列表（根据 evaluationLayer 联动显隐）
   - 迭代次数、批次大小（最多8个，上限256）、优先级
   - 数据类型（FP32/FP16/BF16/INT8 多选）
   - 标签
4. 底部按钮：「克隆为自定义模板」+「关闭」

### 用户故事 B：克隆并修改模板

> 作为验证工程师，我要基于"芯片快速验证"模板，添加 GELU 和 BatchNorm 算子，创建自己的自定义模板。

### 操作闭环

1. 在系统模板详情中点击「克隆为自定义模板」
2. 系统创建副本：POST /api/templates/{id}/clone → 名称 "芯片快速验证 (副本)"、isSystem=false
3. 列表刷新，出现新的自定义模板
4. 点击新模板的「编辑」图标
5. 在编辑 Modal 中修改算子列表（添加 GELU、BatchNorm）
6. 提交 → PUT /api/templates/{id}

### API 详细参数

**POST /api/templates/{id}/clone**

- 无请求体
- 响应：克隆后的模板完整数据
- 命名规则：(副本)、(副本 2)、(副本 3)...

**configJson 标准格式（7个字段）：**

```json
{
  "operators": ["MatMul", "Conv2D", ...],     // string[]，算子级/芯片级必填
  "models": ["ResNet-50", "BERT-Base", ...],  // string[]，模型级/芯片级必填
  "iterations": 100,                           // number，默认100
  "batchSizes": [1, 4, 8],                    // number[]，最多8个，上限256
  "dataTypes": ["FP32", "FP16"],              // string[]，可选FP32/FP16/BF16/INT8
  "priority": "NORMAL",                        // enum: LOW/NORMAL/HIGH/CRITICAL
  "tags": ["推理", "PyTorch"]                  // string[]
}
```

## 3. 评测任务模块

### 用户故事：创建并执行评测任务

> 作为验证工程师，我要针对 Intel Xeon CPU 芯片，使用"算子性能基准测试"模板，在 dev-node-01 节点上执行一次快速评测，并查看完整的评测报告。

### 操作闭环（6步向导）

1. **选择芯片** — 从芯片列表选择目标芯片
2. **选择模板** — 选择评测模板，预览配置
3. **选择评测项** — 根据模板自动填充算子/模型列表，可增删
4. **配置参数** — 选择预设方案（QUICK/STANDARD/FULL/COMPREHENSIVE）
5. **选择节点** — 选择在线的计算节点
6. **确认提交** — 预览摘要，选择"立即执行"或"保存草稿"

### 启动执行流程（TaskDispatcher 链式分发）

1. 用户点击「启动」→ PUT /api/plans/{id}/start
2. 后端：Plan 状态 → RUNNING
3. TaskDispatcher 异步分发第1个 PENDING 任务到 Agent
4. Agent 执行 eval-scripts/cpu_operator_benchmark.py（真实执行）
5. Agent 完成后 POST /api/tasks/{id}/result 回报结果
6. 后端：Task → COMPLETED，更新进度，写入日志
7. TaskDispatcher 自动分发第2个任务（链式）
8. 重复直到全部完成 → Plan → COMPLETED → 自动生成报告

### API 详细参数

**POST /api/plans — 创建评测任务**

```json
{
  "name": "CPU快速算子性能基准测试",   // 必填 string
  "chipId": 449,                        // 必填 long
  "nodeId": 2,                          // 选填 long（不填则手动分配）
  "templateId": 3,                      // 选填 long
  "preset": "QUICK",                    // 选填 enum: QUICK/STANDARD/FULL/COMPREHENSIVE
  "description": "描述"                 // 选填 string
}
```

**PUT /api/plans/{id}/start — 启动任务**

- 前置条件：status=DRAFT
- 响应：更新后的 plan 数据（status=RUNNING）
- 副作用：触发 TaskDispatcher.dispatchNextTask()

**POST /api/plans/{id}/copy — 拷贝任务**

- 复制所有配置 + 自动拆分子任务
- 名称加 "(副本)" 后缀
- 新任务状态 DRAFT

**GET /api/plans/{id}/tasks — 查询子任务列表**

```json
{
  "code": 0,
  "data": [
    {
      "id": 123,
      "taskNo": "TASK-xxx",
      "testSubject": "OPERATOR",
      "testItem": "MatMul",
      "status": "COMPLETED",
      "progress": 100,
      "dimension": "计算性能"
    }
  ]
}
```

## 4. 节点管理模块

### 用户故事：注册并监控计算节点

> 作为运维工程师，我要注册一个新的计算节点，确认 Agent 正常运行，监控节点的在线状态和资源使用情况。

### 操作闭环

1. 在计算节点上部署 Agent（安装依赖、配置 config.yaml、启动 main.py）
2. Agent 自动注册到平台：POST /api/nodes
3. Agent 每 30 秒发送心跳：POST /api/nodes/{id}/heartbeat
4. 用户在「节点管理」页面看到节点状态为 ONLINE
5. 如果节点离线（3分钟无心跳），状态自动变为 OFFLINE
6. 用户点击「诊断」→ POST /api/nodes/{id}/diagnose
   - 返回连通性检查结果、Agent 进程状态、心跳间隔、修复建议
7. 用户点击「修复」→ POST /api/nodes/{id}/repair
   - 尝试 SSH 重启 Agent

### API 详细参数

**POST /api/nodes — 注册节点**

```json
{
  "name": "dev-node-01",                // 必填，唯一
  "ipAddress": "39.97.251.94",          // 必填
  "agentPort": 8090,                    // 选填，默认8090
  "description": "开发测试节点",         // 选填
  "tags": "cpu,dev",                    // 选填
  "sshUser": "root",                    // 选填
  "sshPort": 22                         // 选填
}
```

**POST /api/nodes/{id}/heartbeat — 心跳**

```json
{
  "cpu_percent": 23.5,
  "memory_used_percent": 45.2,
  "memory_available_gb": 8.1,
  "disk_used_percent": 57.3,
  "load_1m": 0.42,
  "timestamp": 1775482000
}
```

**POST /api/nodes/{id}/diagnose — 诊断**

响应：

```json
{
  "code": 0,
  "data": {
    "connectivity": "reachable",
    "agentRunning": true,
    "lastHeartbeat": "2分钟前",
    "issues": [],
    "suggestions": []
  }
}
```

## 5. 资源池管理模块

### 用户故事

> 作为运维工程师，我要创建一个 "CPU 测试池"，将 dev-node-01 节点加入，随时查看资源池的 CPU/内存/GPU 使用情况。

### 操作闭环

1. 进入「资源池」页面
2. 点击「创建资源池」→ 填写名称、类型、描述
3. 在资源池详情中点击「添加节点」→ 选择已注册的节点
4. 查看统计：GET /api/resource-pools/{id}/stats → 总CPU/已用/空闲、总内存/已用/空闲、GPU 数量
5. 当评测任务运行时，对应节点标记为 BUSY，统计中 usedCpuCores 增加

## 7. 执行日志模块

### 用户故事

> 作为验证工程师，我要查看正在执行的任务的实时日志，以及已完成任务的历史日志。

### 操作闭环

1. 在评测任务详情中，点击某个子任务的「日志」按钮
2. GET /api/tasks/{taskId}/logs → 返回日志列表
3. 已完成任务：显示完整的执行日志（stdout + stderr + 指标）
4. 正在执行的任务：当前显示最后已知日志（后续规划 WebSocket 实时推送）
5. 点击「下载日志」→ GET /api/tasks/{taskId}/logs/download → 文本文件

### API 详细参数

**GET /api/tasks/{taskId}/logs**

```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "taskId": 123,
      "logLevel": "INFO",
      "message": "任务开始执行",
      "details": {"operator": "MatMul", "duration_sec": 5.2},
      "createdAt": "2026-04-06T20:55:00Z"
    }
  ]
}
```
