## 一、产品定位

### 1.1 一句话定义

**AHVP 是一个以芯片为中心的 AI 硬件评测平台，通过标准化的算子测试和模型测试，生成芯片能力评价报告，帮助国产芯片厂商、框架厂商和算力采购方回答"这颗芯片适合什么"。**

### 1.2 核心差异化

| 维度 | 竞品现状 | AHVP 差异 |
|------|---------|----------|
| 组织方式 | 以"测试任务"为中心 | **以"芯片档案"为中心**，评测结果自动沉淀 |
| 交付物 | 散落的测试日志/Excel | **自动生成芯片评价报告**（雷达图+评分+建议）|
| 使用方式 | CLI/脚本/线下 | **Web 可视化 + 在线管理** |
| 对标体系 | 各自为政 | **对齐 DeepLink 评测方法学**（算子分类/精度标准/数据收集表）|

### 1.3 目标用户

| 角色 | 核心诉求 | 频率 |
|------|---------|------|
| 芯片厂商评测工程师 | 高效完成数百测试项，生成可交付报告 | 高（日常）|
| 芯片厂商产品经理 | 直观看到能力画像，定位短板 | 中（每周）|
| 框架适配工程师 | 知道哪些算子 fallback、差距多大 | 中 |
| 算力采购决策者 | 性能/能效对比报告 | 低（项目制）|

## 二、核心业务流程

### 2.1 主流程闭环

```
注册芯片 → 创建评测计划 → 执行评测 → 生成评价报告 → 芯片档案沉淀
```

### 2.2 流程详述

```
┌─────────────────────────────────────────────────────────────────┐
│                    AHVP 核心业务闭环                              │
│                                                                   │
│  ① 注册芯片         ② 创建评测计划       ③ 执行评测             │
│  ┌──────────┐       ┌──────────────┐     ┌──────────────┐       │
│  │ 芯片型号  │──────▶│ 选择评测项    │────▶│ 自动调度执行  │       │
│  │ 厂商信息  │       │ 配置参数     │     │ 实时监控进度  │       │
│  │ 技术规格  │       │ 选择节点     │     │ 收集结果数据  │       │
│  └──────────┘       └──────────────┘     └──────┬───────┘       │
│                                                   │               │
│  ⑤ 芯片档案沉淀     ④ 生成芯片评价报告           │               │
│  ┌──────────────┐   ┌──────────────────┐         │               │
│  │ 能力画像更新  │◀──│ 能力雷达图        │◀────────┘               │
│  │ 历史数据归档  │   │ 适用场景推荐      │                        │
│  │ 版本对比基线  │   │ 瓶颈分析+建议    │                        │
│  └──────────────┘   └──────────────────┘                        │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 关键场景（User Story Map）

| 场景 | 用户操作 | 系统行为 | 输出 |
|------|---------|---------|------|
| S1 注册芯片 | 填写名称/厂商/类型 + 可选技术规格 | 生成芯片编号，创建空白档案 | 芯片记录 |
| S2 创建评测计划 | 选芯片→选预设/自定义评测项→配参数→选节点→确认 | 自动拆分 N 个评测任务 | 评测计划 + N 个任务 |
| S3 执行评测 | 查看进度/日志，重试失败任务 | Agent 调度执行，实时上报 | 各任务结果数据 |
| S4 查看报告 | 查看雷达图/排行表/瓶颈分析 | 聚合结果，计算评分，生成报告 | 芯片评价报告 |
| S5 芯片对比 | 选多个芯片横向对比 | 雷达图叠加 + 维度对比表 | 对比视图 |
| S6 查看 Dashboard | 查看芯片统计、评测动态 | 汇总芯片/计划/报告数据 | 运营概览 |

## 三、模块架构

### 3.1 导航结构（4+1）

从原有 13 个模块重组为：

```
🏠 Dashboard                ← 总览仪表盘
💎 芯片管理                  ← 核心入口
   ├── 芯片列表
   ├── 芯片档案（点进列表项）
   └── 芯片对比
📋 评测计划                  ← 创建和管理评测
   ├── 计划列表
   ├── 创建评测计划（5步向导）
   └── 执行监控
🖥️ 节点管理                  ← 计算节点
⚙️ 系统设置                  ← 用户管理 / 操作审计 / 系统配置
```

### 3.2 模块职责划分

| 模块 | 职责 | 核心页面 | 详细设计文档 |
|------|------|---------|------------|
| **M1 芯片管理** | 芯片 CRUD、档案、对比 | 芯片列表、注册页、档案页(4Tab)、对比页 | `module-chip-management.md` |
| **M2 评测计划** | 计划创建向导、预设方案、任务拆分 | 计划列表、创建向导(5步)、计划详情 | `module-eval-plan.md` |
| **M3 评测执行** | 任务调度、Agent 交互、结果收集 | 执行监控、任务详情 | `module-eval-execution.md` |
| **M4 报告引擎** | 数据聚合、评分算法、报告生成 | 评价报告页 | `module-report-engine.md` |
| **M5 Dashboard** | 统计汇总、评测动态、快速操作 | Dashboard 页 | `module-dashboard.md` |
| **M6 节点管理** | 计算节点注册、状态监控 | 节点列表页 | `module-node-management.md` |

## 四、核心实体模型

### 4.1 实体关系概览

```
Chip (芯片)
 │ 1:N
 ├── EvaluationPlan (评测计划)
 │    │ 1:N
 │    └── EvaluationTask (评测任务)
 │         │ 1:1
 │         └── EvaluationResult (评测结果)
 │
 └── ChipReport (芯片评价报告) ←── 聚合自 EvaluationPlan
```

### 4.2 实体简述

| 实体 | 说明 | 核心字段 |
|------|------|---------|
| **Chip** | 平台核心管理对象 | chipNo, name, vendor, chipType, status, specs, profileData |
| **EvaluationPlan** | 一次完整评测活动 | planNo, chipId(必填), preset, status, totalTasks, progress |
| **EvaluationTask** | 最小执行单元 | planId(必填), chipId, testSubject(OPERATOR/MODEL), testItem, dimension, status |
| **EvaluationResult** | 单个任务输出 | taskId, metricsSummary(latency/throughput), status(PASS/FAIL/WARNING) |
| **ChipReport** | 聚合报告（最终交付物） | chipId, planId, overallScore(0-100), dimensionScores, radarData, bottleneckAnalysis |

### 4.3 关键设计原则

1. **任务必须属于计划，计划必须关联芯片** — 不允许孤立任务/报告
2. **testSubject 替代 eval_type** — 只有 OPERATOR/MODEL 两种，解决原有枚举混用
3. **chip_id 冗余存储** — Task 和 Result 中冗余存储，避免多表 JOIN
4. **报告关联 Plan 而非 Task** — 报告是一次评测计划的综合产物

## 五、评测能力体系

### 5.1 评测预设方案

| 预设 | 适用场景 | 包含内容 | 预估任务数 | 预估耗时 |
|------|---------|---------|-----------|---------|
| 🚀 QUICK | 首次接入、日常冒烟 | 5 核心算子 + MLP 推理 | ~7 | ~10min |
| 📋 STANDARD | 版本迭代、常规评测 | 13+ 算子(功能+性能) + MLP(多Batch) | ~17 | ~40min |
| 🔬 COMPREHENSIVE | 季度评测、芯片定级 | 100+ 算子 + 24 模型 + 通信测试 | ~300+ | ~4h |

### 5.2 六维能力评分

| 维度 | 数据来源 |
|------|---------|
| 计算性能 | MatMul/Conv2D 等计算密集型算子 |
| 访存性能 | Transpose/Embedding 等访存算子 |
| 数学函数 | 激活函数类 (ReLU/GELU/SiLU/Sigmoid) |
| Attention 能力 | ScaledDotProduct 等 |
| 归一化性能 | BatchNorm/LayerNorm/RMSNorm |
| 模型推理 | MLP 推理吞吐量 |

### 5.3 DeepLink 对齐

| DeepLink 测试项 | AHVP 对应 | 阶段 |
|----------------|----------|------|
| 算子功能 (100+ 精度) | 算子功能测试模块 | MVP(CPU) / Phase 2(GPU) |
| 算子性能 (GEMM/Conv2d) | 算子性能测试模块 | MVP(CPU) / Phase 2(GPU) |
| 模型功能+性能 (24 模型) | 模型评测模块 | MVP(MLP) / Phase 2(全量) |
| 通信性能 (AllReduce) | 通信性能测试 | Phase 2/3 |
| 大模型评测 | 大模型训练/推理 | Phase 3 |

## 六、分期交付

### MVP-0（核心闭环）

**目标：** 注册芯片→创建评测计划→执行→生成报告，全流程跑通。

| 功能 | 优先级 |
|------|--------|
| 芯片注册 CRUD | P0 |
| 芯片列表（搜索/筛选） | P0 |
| 芯片档案页 (4 Tab) | P0 |
| 创建评测计划 (5步向导 + 3种预设) | P0 |
| 任务自动拆分 + 调度执行 | P0 |
| 执行监控页 | P0 |
| 芯片评价报告自动生成 | P0 |
| 六维评分 + 雷达图 | P0 |
| Dashboard (简化版) | P1 |
| 导航重构 (4+1) | P0 |

**验收标准：**
- ✅ 注册→评测→报告 全流程可走通
- ✅ 评价报告含雷达图+评分+算子排行
- ✅ CPU 环境下数据真实（NumPy 脚本）

### MVP-1（体验完善）

| 功能 | 说明 |
|------|------|
| 芯片档案完整版 | 4 Tab 切换、能力画像默认激活 |
| 能力画像雷达图增强 | 维度评分条 + 适用场景分析 |
| 完整评价报告 | 5 板块（总览/算子/模型/瓶颈/环境） |
| 芯片对比 | 多芯片雷达图叠加 + 维度对比表 |
| 报告 PDF 下载 | 导出为 PDF |

### Phase 2（能力扩展）

100+ 算子精度、24 模型、通信测试、基准值管理、DeepLink 数据收集表导出

### Phase 3（GPU 真机）

GPU/NPU 评测、国产芯片适配(DIPU/DIOPI)、大模型评测、多节点调度
