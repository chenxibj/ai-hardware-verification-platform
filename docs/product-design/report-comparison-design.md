# 评测报告对比功能设计文档

> **版本：** v1.0  
> **日期：** 2026-04-15  
> **作者：** 菜菜子（产品经理 + 架构）  
> **背景：** 从单个报告中移除评分/对比后，需要一个独立的报告对比功能承载 vs 百分比、八维雷达图等比较分析能力。

---

## 一、产品定位

**核心价值：** 让用户选择 2-5 份评测报告，并排对比芯片性能差异，辅助采购决策和性能优化。

**与单个报告的分工：**

| | 单个报告 | 报告对比 |
|---|---|---|
| 展示数据 | 原始测试指标（延迟、吞吐、P95/P99、通过率） | vs 百分比、差值、趋势 |
| 可视化 | 柱状图、表格 | 八维雷达图叠加、差异热力图 |
| 评价体系 | 无评分，只展示事实 | 以基准报告为100%，其他报告算百分比 |
| 适用场景 | 查看单次评测结果 | 芯片选型、版本迭代跟踪、竞品分析 |

---

## 二、用户故事

### US-1：芯片选型对比
**角色：** 算力采购决策者  
**场景：** 有 3 颗候选芯片都跑完了同一套评测模板，需要横向对比选型  
**操作：** 报告列表多选 → 点击 [对比] → 查看对比结果 → 导出 PDF 给领导

### US-2：版本迭代跟踪
**角色：** 芯片厂商研发工程师  
**场景：** 同一颗芯片不同版本（驱动/固件升级前后），对比性能变化  
**操作：** 选同芯片的两份报告 → 查看各指标变化趋势 → 定位退化项

### US-3：基准对标
**角色：** 评测工程师  
**场景：** 把国产芯片和 L40S 基准对标，生成达标率报告  
**操作：** 选基准报告 + 被测报告 → 自动计算 vs 基准百分比 → 导出

## 三、入口与流程

### 3.1 入口（3个）

1. **报告列表页** — 勾选 2-5 份报告 → 顶部出现 [📊 对比分析] 按钮
2. **单个报告详情页** — 右上角 [与其他报告对比] → 弹出报告选择器
3. **芯片详情页** — 该芯片的历史报告列表中多选 → [版本对比]

### 4.3 Section 2：八维雷达图 + 维度对比表

**左侧：八维雷达图叠加**

- 数值为 **vs 基准百分比**（基准 = 100%）
- 如果只有两份报告，用蓝/红；3份加绿；最多5种颜色

**右侧：维度对比表**

|------|:-----------:|:-----------:|:------:|
| 计算 | 100% | **115.2%** 🏆 | 82.3% |
| 访存 | 100% | 95.6% | **103.1%** 🏆 |
| 通信 | 100% | 88.4% | 76.5% |
| 算子兼容 | 100% | 92.1% | 89.7% |
| 训练 | 100% | — | — |
| 推理 | 100% | 91.3% | 85.6% |
| 扩展性 | 100% | 78.9% | 65.2% |
| 生态 | 100% | 71.4% | 57.1% |

**百分比计算公式：**
```
vs_pct(dim) = (基准报告该维度的平均延迟 / 被测报告该维度的平均延迟) × 100%
```
- 延迟更低 → 百分比 > 100%（更快）
- 延迟更高 → 百分比 < 100%（更慢）
- 维度无数据 → 显示 "—"

**颜色规则：**
- ≥ 100%：绿色（达到或超越基准）
- 80-99%：蓝色（接近基准）
- < 80%：红色（低于基准）
- 每行最高值加 🏆

## 五、百分比计算逻辑

### 5.1 算子级百分比

```
vs_pct(operator) = baseline_latency / test_latency × 100%
```

- `baseline_latency` = 基准报告中该算子的 `latencyMean`
- `test_latency` = 被测报告中该算子的 `latencyMean`
- 只有**两份报告都有该算子数据**时才计算，否则显示 "—"

### 5.2 维度级百分比

```
dim_vs_pct(dimension) = avg(vs_pct(op)) for all operators in dimension
```

按维度分组，取组内所有算子 vs 百分比的算术平均值。

### 5.3 综合百分比

```
overall_vs_pct = avg(dim_vs_pct(dim)) for all dimensions with data
```

有数据的维度的平均值。

### 5.4 边界处理

- 基准报告某算子缺失 → 跳过该算子的对比
- 被测报告某算子缺失 → 该算子显示 "—"
- 某维度所有算子都缺失 → 该维度显示 "—"
- 延迟为 0 或负数 → 跳过

## 六、八维雷达图规格

### 6.1 维度定义（与报告一致）

| dimKey | 中文名 | 覆盖的算子 |
|--------|--------|-----------|
| compute | 计算 | MatMul, Conv2D, GEMM, Linear |
| memory | 访存 | Transpose, Embedding, Concat, Gather, Scatter |
| communication | 通信 | AllReduce, AllGather, NCCL, P2P, Broadcast |
| op_compat | 算子兼容 | ReLU, GELU, SiLU, Softmax, LayerNorm, BatchNorm |
| training | 训练 | Backward, Gradient, Optimizer, MixedPrecision |
| inference | 推理 | Attention, ScaledDotProduct, MLP, BERT, LLaMA |
| scalability | 扩展性 | 基于芯片 interconnect 带宽比值 |
| ecosystem | 生态 | 基于芯片 supportedPrecisions 数量比值 |

### 6.2 渲染规格

- **大小：** 400×400px（响应式缩放）
- **网格层数：** 5 层（20%, 40%, 60%, 80%, 100%）
- **基准线：** 100% 处加粗虚线（所有维度的圆上）
- **数据层：** 每份报告一个多边形，半透明填充 + 实线描边
- **颜色：** 蓝 #1890ff / 红 #f5222d / 绿 #52c41a / 橙 #fa8c16 / 紫 #722ed1
- **交互：** hover 显示具体数值 tooltip
- **标签：** 维度名 + 数值（如"计算 115.2%"）

### 6.3 特殊处理

- 维度无数据 → 该维度收缩到 0（雷达图内凹），旁边标注 "无数据"
- 百分比 > 200% → 雷达图裁剪到 200%，数值标签显示实际值
- 单份报告（没有对比对象）→ 不显示雷达图，提示"请选择 2 份以上报告"

## 十一、数据依赖

对比功能完全基于已有的报告数据，不需要新的评测执行：

- `chip_reports.operator_ranking`（JSON）— 算子级原始指标
- `chip_reports.dimension_scores`（JSON）— 维度级评分（后端已有）
- `chip_reports.radar_data`（JSON）— 雷达图数据（后端已有）
- `chips` 表 — 芯片名称、规格信息

**注意：** 当前报告中的 `dimension_scores` 和 `radar_data` 是基于旧版评分逻辑生成的（vs L40S 百分比）。对比功能中，这些评分应该 **重新基于选定的基准报告实时计算**，而不是直接使用 DB 中存储的值。这样才能支持任意报告间的对比，而不仅限于 vs L40S。
