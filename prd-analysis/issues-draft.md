# AI 评测平台 - 需求细化 Issues 草案

基于 PRD 原文 + GitHub 现有 Issues (#29-#34) + 现有系统体验

---

## 一、现有系统体验问题

### 当前系统 (http://39.97.251.94/) 的状态

**已实现的功能框架：**
- 登录/注册
- 工作台（任务统计卡片 + 任务列表）
- 创建评测任务（弹窗形式，字段：任务类型、评测类型、优先级、评测配置JSON）
- 评测报告列表（搜索、创建报告、查看、提交审核、删除）
- 评测编排、报告对比、评测日志、数字资产、计算资源、社区、用户管理、操作审计、系统设置（菜单已有，实现程度待确认）

**发现的核心问题：**

1. **创建任务表单过于简陋**
   - 只有4个字段：任务类型（下拉）、评测类型（下拉）、优先级（下拉）、评测配置（裸JSON输入）
   - PRD 要求的模板选择、评测对象选择、数据集选择、硬件资源配置、batch size、精度模式等全部缺失
   - 普通用户不可能手写 JSON 配置，需要结构化的表单

2. **任务列表缺少「评测对象」列**
   - 表头有"任务类型"但为空，有"评测类型"显示"模型评测/芯片评测"
   - 缺少关键信息：评测对象是什么模型/芯片/框架、用了什么数据集

3. **报告管理缺少版本/模板概念**
   - 目前只有简单的列表+状态流转（草稿→审核→发布）
   - PRD 要求的基础版/高级版区分、报告模板管理、版本控制都没有

4. **缺少任务与报告的关联**
   - 创建报告是独立操作，应该是任务完成后自动生成

## 二、需要新增的 Issues

### Issue 1: 评测任务创建表单重构 — 结构化配置替代裸JSON

**标题**: 【需求细化】评测任务创建表单重构 — 结构化参数配置

**描述**:
当前创建任务只有一个裸JSON输入框，用户体验极差。需要按PRD要求重构为结构化表单。

**功能点**:
- [ ] 任务创建改为分步表单（Step 1: 选择模式 → Step 2: 配置参数 → Step 3: 资源选择 → Step 4: 确认提交）
- [ ] Step 1 — 创建模式选择：模板化创建 / 自定义创建
- [ ] 模板化创建：预置至少3种模板（芯片性能评测、模型推理评测、框架兼容性评测），选择后自动填充默认参数
- [ ] Step 2 — 评测参数配置：
  - [ ] 评测类型选择（性能/精度/兼容性/稳定性）— 下拉框
  - [ ] 评测对象类型选择（芯片/算子/框架/模型）— 下拉框
  - [ ] 评测对象选择（根据对象类型联动，如模型→选择具体模型名称/上传自定义模型）
  - [ ] 数据集选择（预置公开数据集列表 + 自定义上传入口）
  - [ ] 精度模式（FP32/FP16/INT8/BF16）— 多选
  - [ ] Batch Size 配置（输入框，默认值32）
  - [ ] 超时时间设置（分钟，默认60）
- [ ] Step 3 — 资源选择：
  - [ ] 预置资源 vs 自定义资源上传
  - [ ] 硬件资源规格选择（CPU/GPU型号、数量）
  - [ ] 自定义资源上传时自动标记为「自定义评测任务」
- [ ] Step 4 — 确认提交：参数总览 + 提交

**验收标准**:
- 普通用户无需编写JSON即可创建任务
- 模板化创建3步内完成
- 参数校验失败给出明确中文提示

**优先级**: P0
**关联**: #29

### Issue 3: CPU 验收用例定义 — 第一期必须跑通的 Case

**标题**: 【需求细化】CPU 验收用例 — 定义第一期必须在CPU上跑通的评测Case

**描述**:
第一期目标是CPU环境可用，但目前没有定义具体哪些Case必须跑通。需要明确定义验收用例，作为研发交付的验收标准。

**CPU 模型推理验收用例**:
- [ ] Case 1: Qwen-1.8B CPU 推理（FP32），输出首包延迟、平均延迟、吞吐量
- [ ] Case 2: DeepSeek-1.3B CPU 推理（FP32），输出延迟 + 吞吐量
- [ ] Case 3: LLaMA-2-7B CPU 推理（INT8 量化），验证量化后精度损失 ≤ 5%
- [ ] Case 4: ResNet-50 CPU 推理（FP32/INT8），输出图片分类 Top-1/Top-5 准确率

**CPU 算子验收用例**:
- [ ] Case 5: Conv2D 单算子 CPU 执行，不同输入尺寸（224x224, 512x512），输出延迟
- [ ] Case 6: Linear 单算子 CPU 执行，不同矩阵规模，输出延迟
- [ ] Case 7: BatchNorm + ReLU 算子融合 CPU 执行，验证融合 vs 非融合性能差异
- [ ] Case 8: Softmax 单算子 CPU 执行，FP32 vs INT8 精度对比

**验收标准**:
- 以上 8 个 Case 全部在 CPU 环境（Intel Xeon 或同级别）上执行成功
- 输出完整评测报告，包含指标数值 + 可视化图表
- 异常情况（OOM、超时）有明确错误提示和日志

**优先级**: P0

### Issue 4: 评测报告模板细化 — 报告内容结构定义

**标题**: 【需求细化】评测报告模板 — 定义基础版/高级版报告的内容结构

**描述**:
PRD区分了免费基础版和收费高级版报告，但目前没有定义报告具体包含哪些内容。

**基础版报告内容结构（免费，自动生成）**:
- [ ] 报告头信息：报告编号、评测任务编号、评测类型、评测对象、创建时间
- [ ] 评测环境信息：硬件规格（CPU型号、核心数、内存）、软件环境（OS、框架版本）
- [ ] 核心指标结果表：
  - 模型评测：首包延迟、平均延迟、P95延迟、吞吐量、内存占用
  - 芯片评测：算力(TOPS)、能效比(TOPS/W)、利用率
  - 算子评测：执行延迟、吞吐量、精度损失
  - 框架评测：安装成功率、模型加载成功率、兼容性评分
- [ ] 简要结论（自动生成的一句话总结）

**高级版报告内容结构（收费，需申请）**:
- [ ] 包含基础版全部内容
- [ ] 性能趋势图表（延迟趋势、吞吐量变化、资源利用率曲线）
- [ ] 瓶颈分析（如"CPU利用率低，可能是数据加载瓶颈"）
- [ ] 优化建议（具体可落地的方案）
- [ ] 横向对比分析（同类对象对比）

**功能点**:
- [ ] 报告模板数据模型设计
- [ ] 预置3种基础报告模板（模型评测、芯片评测、算子评测）
- [ ] 任务完成后自动生成基础版报告（替代当前手动创建报告）
- [ ] 报告详情页展示结构化内容（不再只是纯文本）
- [ ] 报告导出 PDF/HTML

**优先级**: P0
**关联**: #30

- [ ] 安装成功率(%)
