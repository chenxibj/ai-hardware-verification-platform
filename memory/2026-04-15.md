## 遇到的问题
1. **#444/#445 首次 agent timeout** — 任务量大，第一个 agent 超时，spawn 第二个继续完成
2. **维度 key 双体系** — 旧报告英文 key / 新报告中文 key，对比页需要映射合并
3. **"表现最佳/主要瓶颈"文案在代码中搜不到** — 最终发现是动态生成的边界情况，0 分时 max=min 同一维度

## Lessons Learned
1. **边界值测试很重要** — #440 全零评分是典型边界 case，应在生成报告时就有单测覆盖
2. **维度命名体系要统一** — 前后端+新旧报告用不同 key 是技术债，应在数据层统一而非 UI 层映射
3. **对比功能的核心是公式一致性** — 前后端公式库镜像 + 一致性测试是对的做法，避免计算差异
4. **小 bug 快速修** — #457 从 spawn 到部署 3 分钟，简单 bug 不需要复杂 task prompt
5. **Issue 分组打包效率高** — #452-#456 五个相关 bug 打包一个 agent，6 分钟全部修完
