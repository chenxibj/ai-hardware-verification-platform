## Lessons Learned

### 对比功能串行依赖链效率高
- #444(后端) → #445(前端公式) → #446+#449(入口) → #447+#448(页面+导出) → bug fix
- 严格按依赖链串行 spawn，每步完成才启动下一步，避免并行覆盖
- 全链 ~3 小时完成 6 个 P0 feature

### 新功能上线必然有一波 bug
- 6 个 P0 feature 上线后立刻来了 7 个 bug（#451-#458）
- 经验：大功能上线后预留 1-2 小时 bug fix 窗口，不要急着汇报"全部完成"
- 对比页的 bug 多数是数据兼容性问题（新旧维度体系、JSON 未 parse、边界值），说明 **跨报告数据一致性** 是高风险区

### SummaryCard 组件化是正确做法
- 摘要渲染从 inline JSON.stringify 改为独立 SummaryCard 组件
- 自动字段识别 + camelCase/snake_case/中文兼容 + 兜底展示
- 可复用于其他需要渲染 JSON 摘要的地方
