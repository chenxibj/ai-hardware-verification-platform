## Lessons Learned
1. **搜索 bug 文案先查 DB 再搜代码** — 如果 UI 上显示的文案在代码库 grep 不到，可能是存在 DB 中的旧数据或动态生成的。先 curl API 看实际数据能快速定位。
2. **前端对比页维度映射是必做项** — 后端维度 key 有新旧两套体系（中英文），前端必须有完整映射表。新功能涉及多报告数据时，要考虑历史数据格式兼容。
3. **小 bug 批量打包效率极高** — #452-#456 五个前端 bug 打包给一个 agent，6 分钟搞定。相关 bug 一起修避免多次 build/deploy。
4. **日报 cron 需要检查** — 23:00 日报可能没触发，凌晨补写。需要验证 cron 是否正常。
