### 💡 Lessons Learned
1. **技术债拖延的隐性成本** — Flyway 25 天不做，期间 3 次 schema 相关 bug（4/5 字段缺失、4/27 idx_logs_level、4/29 NOT NULL）。每次应急修复 15-30 分钟，加起来远超一次性做 Flyway 的成本。**技术债应按"累积风险"而非"当前紧急度"排优先级。**
2. **Push 失败 fallback 策略** — sub-agent task prompt 应包含：`如果 git push 超时，将改动保存为 patch 文件（git format-patch），记录到 progressFile，由主 session 接力推送。不要在 push 上无限重试。`
3. **0 open issue ≠ 无事可做（再次验证）** — 今天本可以做 /api/version 构建信息注入、或 review 代码质量、或补测试覆盖。"没有 issue"不是"没有工作"的借口。
