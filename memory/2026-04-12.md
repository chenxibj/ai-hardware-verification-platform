## Lessons Learned
- **Agent 重构产出高效** — #400/#401/#402 三个关联 issue 打包给两个 agent（自愈+批量一组，取消单独一组），13-15 分钟全部完成。关联 issue 打包效率最优。
- **Bug 修复按优先级分组** — P1 两个一组、P2-P3 四个一组，分别 4-8 分钟。小 bug 可以大胆合并。
- **commit 被 sub-agent 推了两次** — 多个 agent 同时操作同一 repo 可能冲突，需要错开部署时机。
- **GitHub close 幂等** — issue 已 close 再 close 不报错，只是提示 already closed，不需要预检。
