- **长期方案：** 需要 mock agent 或将执行测试独立为 @slow 分类

## Lessons Learned

### E2E 测试分层很重要
- **快速 API 测试**（<1s/个）适合 CI 每次 push 跑
- **执行流程测试**（需要 agent 真实处理，3min+/个）应该单独跑，不阻塞 CI
- 混在一起 = CI 要么超时要么太慢，两个都不好

### 版本信息要在部署链路中传递
- Docker build args 只在构建时生效，运行时需要 environment 变量
- `.env` 文件是 docker-compose 传递运行时变量的好方式
- deploy.sh 构建时写 .env → docker compose 读 .env → 容器获得版本信息

### Runner 网络问题处理
- 北京机器连 GitHub 不稳定是常态，需要有自动重试/重启机制
- 当前只能手动 pkill + 重启，考虑加 systemd watchdog
