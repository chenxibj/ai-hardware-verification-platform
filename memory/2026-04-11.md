### 前端 Docker 构建问题（耗时最长的插曲）
- Dockerfile 中 `RUN apk add bash` 在 `RUN npm ci` 之后，但 postinstall 脚本需要 bash → exit 127
- 修复：把 apk add 移到 npm ci 之前
- Alpine 镜像 apk 在 Docker build 中极慢/卡住，最终改用 host 直接 npm build + docker cp 部署
- 来回折腾 5+ 次构建尝试，跨越 3 小时

## Lessons Learned

1. **Dockerfile 指令顺序很重要** — postinstall 脚本依赖的工具必须在 npm ci 之前安装
2. **Alpine apk 在 Docker buildkit 中可能卡住** — 网络问题时考虑 host build + docker cp 作为降级方案
3. **git push 输出到 stderr** — SSH 远程执行时必须 `2>&1` 才能捕获输出
4. **Bug 涌入速度很快** — 今天清完 28 个，晚上又来 13 个新的。说明系统在被积极测试/使用
5. **很多 P1 bug 在修 P0 时顺带修掉了** — 打包修复效率高
