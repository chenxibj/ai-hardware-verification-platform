## Lessons Learned

1. **Alpine Docker build 在中国网络环境下极不稳定** — apk add 会卡死几十分钟。备选方案：host 构建 + docker cp，或换 node:20-slim（debian based）
2. **git push 输出在 stderr** — 必须 `2>&1` 才能捕获，否则看到 exit 0 但无输出
3. **--no-cache 必须在 Dockerfile 有变更时使用** — 否则 buildkit 用缓存的旧 Dockerfile
4. **一天修 28 个 issue 是可行的** — 但需要分批（P0→P1→P2），每批 3-4 个给 sub-agent
5. **很多 P1 bug 会被 P0 修复顺带覆盖** — 先修 P0 再验证 P1 是高效策略
