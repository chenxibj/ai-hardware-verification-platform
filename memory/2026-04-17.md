## 💡 Lessons Learned

### 1. 并行 agent 修不同模块 = 安全；回归 bug 要预期
- 这次 3 个 agent 严格按模块拆分（模板/前端/报告），零冲突零覆盖，验证了"模块隔离并行"策略的有效性
- 但 10 个修复产生了 4 个回归 bug（40% 回归率），说明每轮修复后应预留回归窗口

### 2. Agent timeout ≠ 任务未完成
- fix-report 和 fix-474-477 两个 agent 都超时，但代码修复+部署都已完成
- 超时原因都是"验证阶段耗时过长"，不是"修复未完成"
- **经验：主 session 巡检时不能看到 timeout 就重跑，要先检查代码和部署状态**

### 3. 回归 bug 本质是测试覆盖不够
- #474 回归：模板计数新加 training 但只改了部分组件
- #476 回归：旧报告无 dataStatus 字段，过滤逻辑未兼容
- **根因：修复时的测试只验证了新场景，没覆盖旧数据兼容性**
