## Lessons Learned

- **Code review 是高 ROI 的空闲日活动** — 4 AM 自动 review → 发现 3 个 P0 → 8:40 修完。从发现到修复 <5 小时，完全自主闭环
- **设计文档任务不适合直接 spawn** — agent 会花大量时间探索，容易超时。正确做法：主 session 收集代码结构 → 写入 prompt 或直接写
- **Sub-agent push 必须验证** — 已多次遇到 push 静默失败，巡检逻辑应对比 GitHub HEAD 与开发机 HEAD
- **Agent 的"意外"产出有时是好的** — 编排设计 agent 超时但顺手完成了 Controller 重构和发现已有 Workflows 代码，这些信息比我预设的 task 更有价值
