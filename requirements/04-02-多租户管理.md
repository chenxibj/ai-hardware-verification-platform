# 04-02 多租户管理

**模块**: 用户体系（基础支撑模块）  
**优先级**: P0  
**预计工时**: 25 人日

---

## 功能概述

基于多租户架构，实现租户的创建、管理、隔离，支撑企业、科研机构等多用户群体的协同使用。

---

## 功能点清单

### 04-02-001 租户创建

**描述**: 支持企业用户、科研机构用户、管理员创建租户

**验收标准**:
- [ ] 填写租户信息（租户名称、租户类型、所属行业、联系人、联系方式）
- [ ] 配置租户配额（算力资源配额、存储配额、并发任务配额）
- [ ] 设置租户有效期

**优先级**: P0

---

### 04-02-002 租户类型

**描述**: 支持按行业划分租户类型

**验收标准**:
- [ ] 政务、医疗、工业、金融、科研等类型
- [ ] 不同类型租户可配置不同的默认权限、资源配额

**优先级**: P1

---

### 04-02-003 租户配置

**描述**: 租户创建后的配置管理

**验收标准**:
- [ ] 管理员可修改租户信息、调整资源配额、延长租户有效期
- [ ] 租户创建者可配置租户 logo、租户简介
- [ ] 自定义租户内部组织架构（部门、团队）

**优先级**: P1

---

### 04-02-004 租户状态管理

**描述**: 支持租户状态管理

**验收标准**:
- [ ] 状态：正常、冻结、注销
- [ ] 租户资源配额不足、有效期到期时自动冻结
- [ ] 管理员可手动冻结、解冻租户

**优先级**: P1

---

### 04-02-005 租户注销

**描述**: 支持租户创建者申请注销租户

**验收标准**:
- [ ] 注销前需清理租户内资产、数据
- [ ] 二次确认后完成注销

**优先级**: P2

---

### 04-02-006 数据隔离

**描述**: 不同租户的数据完全隔离

**验收标准**:
- [ ] 评测数据、数字资产、评测报告、日志数据完全隔离
- [ ] 存储在独立的逻辑空间
- [ ] 租户内用户仅可访问本租户的数据
- [ ] 禁止跨租户访问

**优先级**: P0

---

### 04-02-007 资源隔离

**描述**: 按租户配额分配算力、存储资源

**验收标准**:
- [ ] 不同租户的资源相互隔离
- [ ] 租户内用户使用的资源不影响其他租户
- [ ] 租户资源不足时，可申请扩容

**优先级**: P0

---

### 04-02-008 权限隔离

**描述**: 不同租户的用户权限、角色配置相互独立

**验收标准**:
- [ ] 租户内的角色仅可管理本租户的用户、资源、数据
- [ ] 无法操作其他租户的相关内容

**优先级**: P0

---

### 04-02-009 操作隔离

**描述**: 租户内用户的操作仅对本租户可见

**验收标准**:
- [ ] 其他租户无法查看
- [ ] 社区内容可选择"租户内可见"
- [ ] 实现租户内部知识共享

**优先级**: P1

---

### 04-02-010 成员邀请

**描述**: 租户创建者、租户管理员可邀请平台用户加入租户

**验收标准**:
- [ ] 通过手机号、邮箱发送邀请链接
- [ ] 被邀请用户确认后加入租户
- [ ] 可批量邀请多个用户
- [ ] 设置邀请有效期（默认 7 天）

**优先级**: P1

---

### 04-02-011 成员权限分配

**描述**: 为租户内成员分配角色

**验收标准**:
- [ ] 角色：租户管理员、评测专员、资产管理员、普通成员
- [ ] 不同角色对应不同的操作权限

**优先级**: P1

---

### 04-02-012 成员管理

**描述**: 支持查看租户内所有成员列表

**验收标准**:
- [ ] 修改成员角色、移除成员
- [ ] 成员可主动退出租户
- [ ] 退出后不再拥有该租户的相关权限

**优先级**: P1

---

### 04-02-013 部门管理

**描述**: 支持在租户内创建部门、划分团队

**验收标准**:
- [ ] 将成员分配至对应部门/团队
- [ ] 便于租户内组织管理、权限管控
- [ ] 部门管理员可管理本部门的成员

**优先级**: P2

---

- 依赖模块 5-02（资源池管理）提供资源配额管理
