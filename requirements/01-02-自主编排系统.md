# 01-02 自主编排系统

**模块**: 评测系统（核心业务模块）  
**优先级**: P1  
**预计工时**: 30 人日

---

## 功能概述

支持用户可视化编排评测流程，自定义集成算子、脚本，满足复杂评测场景的需求。**本功能所有使用场景均判定为自定义评测任务，按资源消耗收费。**

---

## 功能点清单

### 01-02-001 拖拽式设计器

**描述**: 提供可视化拖拽界面，支持用户添加评测节点、设置节点依赖关系

**验收标准**:
- [ ] 支持拖拽添加评测节点（模型加载、推理测试、精度计算、日志输出等）
- [ ] 支持设置节点依赖关系（如"模型加载完成后再执行推理测试"）
- [ ] 支持节点连线可视化展示流程

**优先级**: P0

---

### 01-02-002 预置节点库

**描述**: 预置常用评测节点，覆盖芯片、算子、框架、模型评测全流程

**验收标准**:
- [ ] 预置至少 10 种常用评测节点
- [ ] 节点分类清晰（芯片评测、模型评测、框架评测等）
- [ ] 支持用户自定义节点

**优先级**: P0

---

### 01-02-003 节点参数配置

**描述**: 每个节点支持自定义输入参数、输出参数、执行条件

**验收标准**:
- [ ] 支持配置输入参数
- [ ] 支持配置输出参数
- [ ] 支持配置执行条件（如"当推理延迟≤100ms 时，执行下一步"）
- [ ] 支持参数联动

**优先级**: P1

---

### 01-02-004 流程预览与调试

**描述**: 支持用户在设计阶段预览评测流程的执行逻辑

**验收标准**:
- [ ] 支持模拟流程执行
- [ ] 提前发现节点配置错误、依赖关系异常
- [ ] 提供错误定位和修改建议

**优先级**: P1

---

### 01-02-005 流程保存与导出

**描述**: 支持用户保存自定义编排的流程，生成流程模板

**验收标准**:
- [ ] 支持保存为流程模板
- [ ] 支持导出流程配置文件（JSON 格式）
- [ ] 支持备份、复用与跨团队分享

**优先级**: P1

---

### 01-02-006 自定义算子上传

**描述**: 支持用户上传自定义算子（Python/C++ 实现）

**验收标准**:
- [ ] 支持上传 Python/C++ 实现的算子
- [ ] 上传时填写算子名称、功能描述、输入输出参数、依赖库
- [ ] 自动校验算子完整性与兼容性

**优先级**: P1

---

### 01-02-007 脚本集成

**描述**: 支持用户上传自定义 Python 脚本，嵌入评测流程的指定节点

**验收标准**:
- [ ] 支持上传 Python 脚本
- [ ] 支持嵌入指定节点
- [ ] 实现个性化评测逻辑（自定义精度计算、异常处理等）

**优先级**: P1

---

### 01-02-008 开发支持

**描述**: 提供算子开发 SDK、脚本开发模板

**验收标准**:
- [ ] 提供算子开发 SDK
- [ ] 提供脚本开发模板
- [ ] 提供开发文档、示例代码

**优先级**: P2

---

### 01-02-009 版本管理

**描述**: 支持自定义算子、脚本的多版本并存

**验收标准**:
- [ ] 自动生成版本号（v1.0、v1.1）
- [ ] 用户可指定评测流程使用的版本
- [ ] 支持版本回滚

**优先级**: P1

---

### 01-02-010 预置流程模板

**描述**: 内置常用评测流程模板

**验收标准**:
- [ ] 预置"芯片性能评测流程"
- [ ] 预置"模型精度验证流程"
- [ ] 预置"框架兼容性评测流程"
- [ ] 用户可直接复用

**优先级**: P1

---

### 01-02-011 自定义模板管理

**描述**: 支持用户将自定义编排的流程保存为模板

**验收标准**:
- [ ] 填写模板名称、描述、适用场景
- [ ] 便于后续快速调用
- [ ] 支持模板检索与删除

**优先级**: P1

---

### 01-02-012 模板分享

**描述**: 支持用户将自定义模板分享给团队成员、同租户用户

**验收标准**:
- [ ] 设置模板查看/编辑权限
- [ ] 提升团队协作效率
- [ ] 分享记录可追溯

**优先级**: P2

---

### 01-02-013 模板版本控制

**描述**: 记录模板的修改历史

**验收标准**:
- [ ] 记录修改人、修改时间、修改内容
- [ ] 支持模板版本回滚
- [ ] 避免误修改导致的模板失效

**优先级**: P2

---

## 依赖关系

- 依赖模块 1-01（评测任务管理）执行编排后的任务
- 依赖模块 2-04（数字资产管理）存储流程模板

## 接口需求

- 流程编排 API
- 算子上传 API
- 模板管理 API
